# 2D Medical Image Generation Thesis Design

## Topic

Research design for a thesis on 2D medical image generation, self-supervised learning, and downstream 2D medical image segmentation, with generation training as the primary task and SSL plus segmentation used as the evaluation chain.

## Project Context

The current repository already provides an end-to-end experimental pipeline for:

- SSL pretraining under `2d-ssl-seg/src/run_ssl_pretrain.py`
- encoder extraction under `2d-ssl-seg/src/extract_backbone.py`
- downstream segmentation training under `2d-ssl-seg/src/train_segmentation.py`
- segmentation evaluation under `2d-ssl-seg/src/evaluate_segmentation.py`

Existing experiments already cover:

- SSL methods: MoCo v3, SimCLR, VICReg
- segmentation models: ResNet18 encoder + UNet decoder, and pure UNet

This means the repository already supports the back half of the research chain, namely "pretraining -> downstream segmentation evaluation". What is not yet defined as a thesis structure is the front half: generation training as the main research object, plus a clear experimental matrix and proposal narrative around synthetic data value.

## Thesis Positioning

This thesis should be positioned as a generation-driven pretraining data study rather than a pure SSL thesis or a pure segmentation thesis.

The primary task is generation model training and generation experiments. SSL is the bridge that converts generated images into learned representations. Downstream segmentation performance is the final evidence used to determine whether generated data has practical value.

The core workflow is:

1. Fine-tune multiple text-to-image generation base models on the same medical dataset with LoRA.
2. Generate synthetic medical images from each fine-tuned model.
3. Use those generated images as pretraining data for VICReg.
4. Transfer the pretrained encoder into a fixed downstream segmentation pipeline.
5. Compare final segmentation performance to determine whether stronger generation models produce more valuable pretraining data.

## Research Questions

### Main Research Question

Given the same medical dataset and the same LoRA fine-tuning protocol, can different base text-to-image generation models produce synthetic medical images that lead to different downstream 2D medical image segmentation performance?

### Sub-question 1

Do quality differences between synthetic images generated by different fine-tuned base models propagate into the later pretraining and segmentation stages?

### Sub-question 2

Are generation quality metrics or medical realism assessments aligned with final segmentation performance, or can those two views diverge?

## Hypotheses

### H1

Stronger base text-to-image models, after LoRA fine-tuning on the same medical dataset, can generate more valuable synthetic medical images and thus improve final downstream segmentation performance.

### H2

Improvements in generation quality and medical realism will generally correlate positively with improvements in downstream segmentation performance.

### H3

This relationship can be consistently observed through the chain:

`synthetic data -> VICReg pretraining -> segmentation fine-tuning -> segmentation metrics`

## Scope Boundaries

To keep the thesis focused and experimentally tractable, the study should use one primary variable and several fixed modules.

### Primary Variable

The main experimental variable is the base text-to-image generation model version.

The study compares:

- different base generation models
- the same medical LoRA fine-tuning dataset
- a unified LoRA fine-tuning protocol as far as practical
- the same synthetic dataset construction process
- the same VICReg pretraining process
- the same downstream segmentation evaluation process

### Fixed Decisions

- The medical training dataset is already fixed and is not a research variable.
- Data cleaning strategy is out of scope.
- The primary SSL method is fixed as VICReg.
- The primary downstream segmentation model is fixed as ResNet18 encoder + UNet decoder.
- Pure UNet is not part of the main thesis line and can only appear as a supplementary experiment if time allows.

### Explicit Non-goals

The thesis should not treat the following as primary variables:

- SSL method comparison among MoCo v3, SimCLR, and VICReg
- segmentation architecture comparison
- data cleaning strategy
- prompt engineering as a standalone main topic
- LoRA hyperparameter sweeps as the thesis main line
- synthetic and real data mixing strategies as a main line

These can appear only as supporting observations, supplementary checks, or future work.

## Recommended Research Direction

The recommended design combines two layers:

- Main direction: generation model ranking and comparison
- Supporting direction: analysis of whether generation-side quality signals align with downstream task gains

This means the thesis should primarily ask whether stronger base generation models produce more useful synthetic pretraining data, while secondarily analyzing whether better generation metrics or realism judgments actually explain the downstream results.

This avoids two failure modes:

- becoming a broad SSL method comparison study
- becoming a large synthetic-data mixing study that dilutes the generation-training focus

## Method Design

### 1. Generation Module

Select multiple base text-to-image models. Fine-tune each one on the same medical dataset using LoRA under as consistent a protocol as possible. Export LoRA weights and use each fine-tuned version to generate a synthetic medical image dataset of matched size.

### 2. Synthetic Dataset Construction

For each generation model version, construct a synthetic dataset under a unified generation protocol. The dataset size, output format, and generation procedure should remain fixed across compared versions so that downstream differences are attributable mainly to the base generation model.

### 3. SSL Module

Use the synthetic dataset from each generation model version to run VICReg pretraining. The SSL configuration should remain fixed across experiments.

### 4. Segmentation Module

Extract the pretrained encoder and fine-tune a fixed downstream segmentation model, defined as ResNet18 encoder plus UNet decoder, on the same data split and evaluation protocol.

### 5. Analysis Module

Analyze two result layers:

- final segmentation performance differences across generation model versions
- the relationship between generation-side quality or realism indicators and downstream segmentation outcomes

## Experiment Plan

The experiment plan should proceed in phases so that the thesis reaches a closed research loop early, then expands only if needed.

### Phase 1: Minimum Closed Research Loop

Goal: determine whether generation model version differences can affect final segmentation performance.

Steps:

1. Select a small number of candidate base text-to-image models. The initial recommendation is three models.
2. Fine-tune each base model on the same medical dataset using LoRA.
3. Generate a synthetic dataset of matched scale from each fine-tuned model.
4. Pretrain VICReg on each synthetic dataset.
5. Transfer the pretrained encoder to the fixed segmentation model.
6. Evaluate final segmentation results with Dice, IoU, and HD95.

This phase answers the main thesis question with the smallest viable experimental matrix.

### Phase 2: Required Baselines

At minimum, the study should include these baselines:

1. Random initialization with no SSL pretraining.
2. Real-only VICReg pretraining followed by downstream segmentation.
3. Synthetic-only VICReg pretraining for each compared generation model version followed by downstream segmentation.

An optional extension, if time permits, is:

4. Real plus synthetic pretraining.

This optional mixed-data setting should not be considered required for the first closed-loop thesis milestone.

### Phase 3: Result Interpretation

After the main loop and baselines are completed, add supporting analyses:

1. generation quality and medical realism comparison across model versions
2. correlation analysis between generation-side indicators and downstream segmentation performance

These analyses support the secondary thesis question without expanding the main variable set.

## Main Experimental Matrix

The minimum recommended main matrix is:

- 3 base text-to-image models
- 1 fixed medical LoRA dataset
- 1 unified LoRA fine-tuning protocol
- 1 matched synthetic dataset size per model
- 1 SSL method: VICReg
- 1 primary segmentation model: ResNet18 + UNet
- 3 core baselines: random init, real-only SSL, synthetic-only SSL

This matrix is large enough to support a thesis argument but still controlled enough to remain coherent.

## Proposal Structure

### 1. Introduction

Explain why generated medical images should not be evaluated only by visual quality, but also by whether they create value as pretraining data for downstream tasks. Motivate the problem through medical data scarcity and the need for stronger pretraining resources.

### 2. Related Work

Review three areas:

- medical image generation
- self-supervised learning in medical imaging
- synthetic data for downstream segmentation

The gap to highlight is that existing work often studies generation quality or downstream utility separately, but less often studies whether different fine-tuned generation model versions systematically alter downstream segmentation performance through a fixed pretraining chain.

### 3. Methodology

Present the complete pipeline:

- LoRA fine-tuning of multiple base generation models
- synthetic dataset construction
- VICReg pretraining
- downstream segmentation fine-tuning and evaluation

This section should emphasize control of variables across the compared generation models.

### 4. Experimental Design

Specify:

- datasets
- compared generation models
- baseline settings
- evaluation metrics
- implementation protocol

This section should make it clear that the experimental design is controlled and interpretable.

### 5. Planned Analysis

Focus on:

- comparing final segmentation performance across generation model versions
- analyzing whether generation-side quality signals align with downstream performance

### 6. Timeline / Research Plan

Organize the work into:

- Phase 1: generation fine-tuning plus synthetic dataset construction
- Phase 2: VICReg pretraining plus main segmentation loop
- Phase 3: baselines plus result analysis
- Phase 4: supplementary experiments and thesis writing

## Framing Rules For Writing

To keep the thesis narrative stable, these statements should remain consistent throughout the proposal and later thesis drafts:

- generation training and generation experiments are the primary task
- SSL is the bridge between synthetic data and learned representation
- downstream segmentation performance is the main evidence of practical value

The thesis should not be framed as asking which SSL method is best or which segmentation architecture is best. Its actual question is whether different fine-tuned generation model versions produce synthetic data with different pretraining value, and whether that value appears in final segmentation performance.

## Self-review

This design is scoped to one main thesis line. It avoids placeholder sections, keeps the primary variable explicit, and does not expand into unrelated axes such as data cleaning, broad SSL comparison, or large segmentation architecture sweeps. The recommended experiment plan reaches a closed loop early and supports later analysis without requiring a full combinatorial study.
